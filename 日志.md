### 16:37 — 设计系统硬约束提权

**调整内容：** 把卡片留白、模块间距、品牌 Logo 使用、表格圆角四项规则从"建议"提升为"强约束"，使用设计体系时必须第一时间遵守。

**动机：** 在使用过程中发现：相同的规则若只是描述性地写在文档里，下游生成的页面会出现卡片留白 24/16/12 混用、模块间距随手写、Logo 直接被替换成纯文字、表格被加上 8px / 12px 圆角等漂移问题。需要在文档中明确"非协商"，让设计体系本身具备拒绝偏离的能力。

**具体操作：**
- `DESIGN.md` — 新增四段强调式约束：
  - **Card padding enforcement**：所有卡片组件内边距必须 20px (`var(--space-20)`)。
  - **Module spacing enforcement**：模块/卡片/区段之间默认间距必须 12px (`var(--space-12)`)，`--space-24` 仅作为显式覆盖项。
  - **Brand asset enforcement**：所有顶部导航、侧栏、应用外壳必须渲染对应品牌 Logo 变体，禁止用纯文字 / 占位 SVG / 省略代替。
  - **Wrapper-padding enforcement**：`.table-wrap` 自带 20px 内边距，禁止再嵌套一层 `padding`，避免双层留白破坏对齐。
- `DESIGN.md` 反模式章节同步加入对应"❌"项，使违例点可被自检。
- `colors_and_type.css` — 把 `.table-wrap` 的圆角强制为 0px，并内置 `padding: 20px`。
- `preview/components-cards.html` / `components-tables.html` / `components-pagination.html` / `spacing-tokens.html` 等同步对齐到新约束。
- `AGENT.md` / `SKILL.md` 中重申约束优先级。

## 2026年06月15日

### 14:07 — 主题系统提权 + 卡片层级语义化（规范沉淀）

**调整内容：** 在原有单一蓝色主题之上，引入「博瑞蓝 / 国网绿 / 南瑞蓝」三套主题色系，并把卡片样式从单一 `.card` 拆分为「一级卡片（无边框，结构性模块用）/ 二级卡片（带边框，列表/嵌套用）」的语义化双轨。

**动机：**
- 设计系统要服务多个品牌方（博瑞、国网、南瑞），原本只有一套蓝色主色无法承接。原来的做法是每次复制一份 token 文件改主色，但这样语义、间距、字体规则会逐渐分叉，无法长期维护。
- 卡片在使用中暴露出"什么时候带边框、什么时候不带"的判断模糊：仪表盘 KPI、Usage Guidelines 等结构性模块被错误地包了 `border:1px solid var(--border-color)`，与浅灰蓝页面背景的视觉分隔重复打架，整体显得碎。需要把"结构性模块用一级、嵌套/列表项用二级"写成系统规则。

**具体操作：**

1. **多主题体系（`DESIGN.md` §2 + `colors_and_type.css`）**
   - 新增「Theme System」章节：三套主题共享语义、背景、文字、边框、字体、间距、圆角、阴影、动效全部 token，仅主色组（`--color-primary` / `--color-primary-hover` / `--color-primary-light` / `--color-primary-bg`）+ 派生的导航渐变 + 侧栏选中色四组随主题切换。
   - 主题通过 `<html data-theme="blue|green|nari">` 切换；`colors_and_type.css` 里以 `[data-theme="green"]` / `[data-theme="nari"]` 作用域覆写主色组。
   - 新增 `--color-primary-active` 变量（蓝 `#0A64EA` / 绿 `#00806B` / 南瑞 `#002A55`）作为顶部导航激活态背景，确保激活项在渐变条上仍然可读；`--sidebar-active-bg` 保留为旧别名以兼容 v1.0 消费方。

2. **主题选择优先级规则**
   - 写入「Theme Selection — first-step rule」：每个新产品在做布局之前，必须先回答主题色选择题（a 博瑞蓝 / b 国网绿 / c 南瑞蓝 / d 多主题带切换器），并把答案落到 `<html data-theme="…">`。
   - 支持两种使用模式：构建期锁定（不出切换器）和运行时换肤（带 `.theme-switcher`，选择持久化到 `localStorage:joinbright:theme`）。

3. **`.theme-switcher` 切换器组件（`DESIGN.md` §6.1 + `colors_and_type.css`）**
   - 顶部导航右侧、`.nav-actions` 之前的下拉选择器：触发按钮 32px 高、6px 圆角，背景固定 `#ECF7FF`（关闭态稳定性优先），文字主题感知。
   - 菜单：白底、`--shadow-base`、8px 圆角、4px 内边距、160px 最小宽度；每行 32px 高、6px 圆角、带 10px 圆形主色色块。
   - 行为：点击触发 `data-theme` 切换 + `localStorage` 持久化；点击外部 / `Esc` 关闭；脚本统一收口在 `colors_and_type.css` 同捆的 IIFE 中，复制即用。
   - 在 `preview/components-navigation.html` 中加入主题切换器示例与多主题导航条对照。

4. **卡片分级（`DESIGN.md` §6.11）**
   - 明确「一级（`.card`，无边框）」用于：仪表盘面板、KPI 数据卡、章节容器、Usage Guidelines / 信息块、任何锚定一个页面区域的独立结构模块 — 由 `--bg-page` 浅灰蓝背景提供视觉分隔。
   - 明确「二级（`.card-secondary`，1px 边框）」用于：卡片列表项、嵌套设置组、模态内卡片等需要在一级模块内部再做边界的同级单元。
   - 在反模式章节加入"❌ 把结构性模块包成 `border:1px solid var(--border-color)`"，与卡片留白 20px、模块间距 12px 同级强约束。
   - 同步刷新 `preview/components-cards.html`，新增对照样例。

5. **预览页规范层定调（本时段仅约束层落定，实际页面同步在 15:50 完成）**
   - 在 `DESIGN.md` §6.1 增补 `.theme-switcher` 文档；为后续预览页改造提供可对照的契约。
   - 同步沉淀 `日志.md`（本文件），把 06月08日 三段历史记录与本日规范层调整一并入档。

### 16:13 — `colors_and_type.css` 主题层 + 切换器样式 + 脚本同捆

**调整内容：** 把多主题的实际 CSS / JS 实现一次性收口到 `colors_and_type.css`，让消费方只引一份样式表就拿到完整的主题系统能力。

**动机：** 早期把切换器脚本散落在使用页面里，复制成本高且容易漂移；统一收口后下游引用一个 `.css` 即可拥有 token + 切换器样式 + 切换 IIFE 脚本的全套能力。

**具体操作：**
- 在 `colors_and_type.css` 顶部 `:root` 之外增补 `[data-theme="green"]` / `[data-theme="nari"]` 作用域，覆写 `--color-primary` 系列 + `--nav-gradient-from/to` + `--color-primary-active`。
- 加入 `.theme-switcher` / `.theme-switcher__trigger` / `.theme-switcher__menu` / `.theme-switcher__option` / `.theme-switcher__swatch--{blue,green,nari}` / `.theme-switcher__caret` 全套样式（关闭态背景固定 `#ECF7FF`，hover/open 文字色随主题感知）。
- 同捆切换器行为脚本（IIFE）：点击触发 → 切换 `data-theme` + `localStorage:joinbright:theme` 持久化 + 同步所有切换器 UI；外部点击 / `Esc` 关闭；页面加载自动恢复 saved theme。
